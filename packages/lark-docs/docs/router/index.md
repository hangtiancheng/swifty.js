# Router

`@lark.js/docs` generates file-based routes from your `docs/` directory and maps them to `@lark.js/mvc` Router routes. The Router supports both `history` mode (clean URLs) and `hash` mode (fragment URLs).

## Route Generation Rules

Each `.md` file in the docs directory becomes one route:

| File Path                 | Generated Route         | View ID           |
| ------------------------- | ----------------------- | ----------------- |
| `docs/index.md`           | `/docs/`                | `index`           |
| `docs/getting-started.md` | `/docs/getting-started` | `getting-started` |
| `docs/guide/index.md`     | `/docs/guide/`          | `guide-index`     |
| `docs/guide/config.md`    | `/docs/guide/config`    | `guide-config`    |
| `docs/api/router.md`      | `/docs/api/router`      | `api-router`      |

Rules:

- `index.md` maps to the directory root (with trailing `/`)
- Other files map to their stem without `.md` extension
- All paths are prefixed with `baseUrl`
- Files/directories starting with `_` or `.` are ignored
- `node_modules`, `__tests__`, `.git`, `dist` directories are skipped

## Route Modes

### History Mode (Default)

Uses `history.pushState` and `popstate` events for clean URLs:

```
https://example.com/docs/guide/
https://example.com/docs/guide/config
https://example.com/docs/api/router
```

```ts
routeMode: "history";
```

::: tip
History mode requires server-side configuration to serve the same HTML file for all routes (a "catch-all" or "SPA fallback"). Most static hosting services support this out of the box.
:::

### Hash Mode

Uses URL hash fragments with the `#!` prefix:

```
https://example.com/#!/docs/guide/
https://example.com/#!/docs/guide/config
https://example.com/#!/docs/api/router
```

```ts
routeMode: "hash";
```

::: warning
Hash mode does not work with server-side rendering. It is best suited for static hosting where server configuration is not available.
:::

## Base URL

The `baseUrl` config option adds a common prefix to all routes:

```ts
baseUrl: "/docs/"; // /docs/, /docs/guide/, /docs/api/router
baseUrl: "/v2/docs/"; // /v2/docs/, /v2/docs/guide/
baseUrl: "/"; // /, /guide/, /api/router
```

## Route Map

The generated route map is a plain object that feeds directly into `@lark.js/mvc`'s `FrameworkConfig.routes`:

```ts
// Generated by vite.config.ts
export const routes = {
  "/docs/": "index",
  "/docs/get-started/": "get-started-index",
  "/docs/get-started/configuration": "get-started-configuration",
  "/docs/markdown/": "markdown-index",
  "/docs/markdown/frontmatter": "markdown-frontmatter",
};
```

## Navigation

### In Markdown

Links starting with `/` or `#` are automatically intercepted for SPA navigation:

```markdown
[Guide](/docs/guide/) → Router.to("/docs/guide/")
[#installation](#installation) → scroll to anchor
[GitHub](https://github.com) → opens in new tab (not intercepted)
```

### Programmatic

In theme views, navigation is delegated to the lark-mvc Router:

```ts
Router.to("/docs/guide/config"); // navigate
Router.to("/docs/guide/", { page: "2" }); // navigate with params
Router.to({ page: "3" }); // update params only
```

## Unmatched Routes

When a URL does not match any generated route, the `unmatchedView` is mounted. Configure this in the boot file:

```ts
Framework.boot({
  routes,
  unmatchedView: "index", // fallback to home page
});
```
