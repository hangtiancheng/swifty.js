# Router

`@lark.js/docs` generates file-based routes from your `docs/` directory and maps them to `@lark.js/mvc` Router routes. All routes map to a single layout view (`theme/docs-layout`) which stays mounted across navigation and swaps content via `loadContent()`.

## Route Generation Rules

Each `.md` file in the docs directory becomes one route:

| File Path                 | Generated Route         |
| ------------------------- | ----------------------- |
| `docs/index.md`           | `/docs/`                |
| `docs/getting-started.md` | `/docs/getting-started` |
| `docs/guide/index.md`     | `/docs/guide/`          |
| `docs/guide/config.md`    | `/docs/guide/config`    |
| `docs/api/router.md`      | `/docs/api/router`      |

Rules:

- `index.md` maps to the directory root (with trailing `/`)
- Other files map to their stem without `.md` extension
- All paths are prefixed with `baseUrl`
- Files/directories starting with `_` or `.` are ignored
- `node_modules`, `__tests__`, `.git`, `dist` directories are skipped

## Route Map

All routes map to a single view ID -- `"theme/docs-layout"`. The layout stays mounted across navigation and loads different content via `loadContent(path)`:

```ts
// Generated by defineConfig() in .lark-docs/generated/index.js
const LAYOUT_VIEW = "theme/docs-layout";

export const routes = Object.fromEntries(
  Object.keys(loaders).map((k) => [k, LAYOUT_VIEW]),
);
// Result:
// {
//   "/docs/": "theme/docs-layout",
//   "/docs/get-started/": "theme/docs-layout",
//   "/docs/get-started/configuration": "theme/docs-layout",
//   "/docs/markdown/": "theme/docs-layout",
//   "/docs/markdown/frontmatter": "theme/docs-layout",
// }
```

This design means the layout view is never unmounted and remounted on navigation. Instead, `observeLocation` triggers an async `render()` that calls `loadContent(path)` to fetch the new page's compiled markdown.

## Route Modes

### History Mode (Default)

Uses `history.pushState` and `popstate` events for clean URLs:

```
https://example.com/docs/guide/
https://example.com/docs/guide/config
https://example.com/docs/api/router
```

```ts
routeMode: "history";
```

::: tip
History mode requires server-side configuration to serve the same HTML file for all routes (a "catch-all" or "SPA fallback"). Most static hosting services support this out of the box.
:::

### Hash Mode

Uses URL hash fragments with the `#!` prefix:

```
https://example.com/#!/docs/guide/
https://example.com/#!/docs/guide/config
https://example.com/#!/docs/api/router
```

```ts
routeMode: "hash";
```

::: warning
Hash mode does not work with server-side rendering. It is best suited for static hosting where server configuration is not available.
:::

## Base URL

The `baseUrl` config option adds a common prefix to all routes:

```ts
baseUrl: "/docs/"; // /docs/, /docs/guide/, /docs/api/router
baseUrl: "/v2/docs/"; // /v2/docs/, /v2/docs/guide/
baseUrl: "/"; // /, /guide/, /api/router
```

## Navigation

### In Markdown

Links starting with `/` or `#` are automatically intercepted for SPA navigation:

```markdown
[Guide](/docs/guide/) -> Router.to("/docs/guide/")
[#installation](#installation) -> scroll to anchor
[GitHub](https://github.com) -> opens in new tab (not intercepted)
```

The markdown compiler adds `data-lark-nav="true"` to internal links. The lark-mvc Router intercepts clicks on these elements for client-side navigation.

### Programmatic

In theme views, navigation is delegated to the lark-mvc Router:

```ts
Router.to("/docs/guide/config"); // navigate
Router.to("/docs/guide/", { page: "2" }); // navigate with params
Router.to({ page: "3" }); // update params only
```

## Unmatched Routes

When a URL does not match any generated route, the `unmatchedView` is mounted. Configure this in the boot file:

```ts
Framework.boot({
  routes,
  unmatchedView: "theme/docs-layout", // fallback to layout with "Page not found"
});
```

The layout view's `loadContent()` returns `null` for unknown paths, causing the layout to render a "Page not found" message.
